Frameworks I know

Below are the frameworks I know.
A framework is something which is premade in a certain language which helps you.

Basically like a huge book of code thats already their to help you out in a language after installing it!

So yeah, these frameworks are very useful. Their language will also be put along side them in this format: Framework | Language as the title!



Bootstrap | CSS, JS

I learned this about 2 months after learning PHP. Since I couldn't code CSS all day and I didn't like coding CSS.

I just liked coding HTML and a bit of JS at that time! So I googled up some code and came to near no-where. Then finally I found bootstrap, I went to w3schools and quickly figured Bootstrap out.

Note: This website is proudly made with Bootstrap.

Here is what we actually use and I haven't coded any of this stuff. Just link bootstrap to my code and I get this stuff:

  • Scrollspy (when you click languages and then HTML scrolling you to HTML)
  • 500+ Classes
  • Colors like teal, blue, green and more!
  • Headers and jumbotrons
  • Container classes (all this column stuff)
  • Custom SASS and JavaScript














W3 CSS | CSS

A css library arguably more powerful than bootstrap itself. It was inspired by bootstrap and the fact that when W3Schools taught HTML and CSS they wrote over a million lines of CSS.

Then with one line you can import that 1 million lines of CSS!
This website uses W3 CSS aswell because I could not figure out which one was more powerful so I choosed both.

I am still exploring the potential of W3 CSS's styling. For JavaScript my main framework is bootstrap still! Note: This website also uses W3 CSS!

Django | Python

I used Django to build a sample site I think django is good, but not nearly as good as some other web developement frameworks. This is because most of it is actually just plain python object oriented programming!

The problem with this is that you would need a lot more modules just to write some authentication / server side.
Your interactions with the background server is only by accessing the server. This not only makes it hard to write SQL queries. But hard in general!

The models are easy to setup and its good, just that there are much better alternatives like ExpressJS.













ExpressJS | JavaScript

I learned ExpressJS as part of the MEAN stack! Yash Bhatnagar my friend has created plenty of projects with ExpressJS and the MEAN stack. Incase you don't know what the MEAN stack is. It is: MongoDB, ExpressJS, AngularJS, NodeJS. Basically it is JavaScript only website. Since MongoDB is a database. Obviously it has some static HTML, CSS, SASS etc.

Now this got cool, NodeJS had a faster but less efficient backend than PHP. Ultimately I chose PHP but I had fun writing ExpressJS code. This is one of the reasons that got me back to JavaScript. NodeJS was another!
